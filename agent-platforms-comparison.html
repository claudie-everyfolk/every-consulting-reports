2. What OpenAI Just Shipped

OpenAI released two related things in the past week. The X post Dan shared is about Workspace Agents — the product layer. But there's also an Agents SDK update (April 15) underneath it.

Workspace Agents (April 22) — the product

Persistent, Codex-powered agents that run in the cloud and keep working when you're not there. Teams build them inside a new ChatGPT tab by describing desired workflows. The system maps processes, connects tools, and tests functionality before activation.

Key capabilities

  • Always-on cloud execution — agents run autonomously, not just when someone's chatting
  • Scheduling and triggers — run on schedule or in response to specific events
  • Integrations — Google Drive, Google Calendar, Slack, SharePoint, custom MCPs
  • Approval gates — agents ask for human approval before sensitive actions
  • Shared across teams — build once, use in ChatGPT or deploy to Slack
  • Org-level admin controls — limit agent access to specific data and tools
  • Memory — retain context across projects
  • Prompt injection safeguards — built-in protections

Availability and pricing

  • Research preview: ChatGPT Business, Enterprise, Edu, Teachers
  • Free through May 6, 2026
  • Credit-based pricing after that (amounts not yet published)
  • Evolution of GPTs — existing GPTs will be convertible to workspace agents later

Agents SDK Update (April 15) — the infrastructure

For developers building their own agent systems via the API. Introduces a harness + sandbox architecture that separates the control plane from the compute plane.

Key additions

  • Harness — model-native control plane: agent loop, tool routing, handoffs, approvals, tracing, recovery, run state
  • Sandbox — isolated execution plane: reads/writes files, runs commands, installs deps, exposes ports, snapshots state
  • Pluggable sandbox providers — Blaxel, Cloudflare, Daytona, E2B, Modal, Runloop, Vercel, or BYO
  • Manifest abstraction — portable workspace definitions. Mount local files, output dirs, cloud storage (S3, GCS, Azure Blob, R2)
  • MCP support — tool integration via Model Context Protocol
  • AGENTS.md — custom instructions file (equivalent to Claude Code's CLAUDE.md)
  • Credential isolation — harness/compute split keeps tokens out of sandboxes
  • Snapshotting + rehydration — restore agent state in fresh containers after failures
  • Subagents (coming) — parallel, isolated task decomposition
  • Python now, TypeScript later. Standard API pricing.
The relationship: Workspace Agents is the product built on top of the Agents SDK. The SDK gives developers the primitives. Workspace Agents packages them into a no-code product for teams. Same pattern as Claude Managed Agents (infrastructure) sitting on top of the Agent SDK (framework).

Security verdict: The cloud platforms (OpenAI Workspace, Claude Managed) are architecturally stronger on isolation — credentials never touch the execution environment. Claude Managed Agents has the most rigorous credential isolation (vault + proxy pattern). OpenAI Workspace Agents has the most accessible approval flows (built into the product). The self-hosted options (OpenClaw, Claudie) trade isolation for capability — full machine access enables things sandboxed agents can't do, but requires compensating controls. Claudie's ring system and PreToolUse hooks are the most sophisticated compensating controls of any platform, but they're defense-in-depth, not structural isolation.

6. Architecture Deep Dive

The converging pattern: brain/hands separation

Both OpenAI and Anthropic independently arrived at the same architectural insight: separate the brain (model) from the hands (execution environment).

OpenAI's framing

Harness = control plane (agent loop, model calls, tool routing, handoffs, approvals, tracing, recovery, run state)

Sandbox = compute plane (file reads/writes, commands, deps, ports, state snapshots)

Anthropic's framing

Harness = stateless loop calling Claude, routing tool calls. Crashes? Reboot, replay from session log.

Sandbox = cattle containers. execute(name, input) → string. Brain doesn't know if sandbox is a container or "a Pokémon emulator."

Why this matters for security: If the brain never enters the sandbox, and credentials live outside the sandbox, then prompt injection that compromises the brain can't exfiltrate credentials from the sandbox. Both platforms use this to structurally eliminate the credential-exposure attack surface rather than relying on scoping rules that degrade as models get smarter.

OpenClaw and Claudie: the no-sandbox approach

Both run on real machines with no isolation boundary. The security model is fundamentally different: instead of preventing the agent from accessing credentials, you trust the agent with credentials and add compensating controls.

OpenClaw's approach

  • Trust model: you installed it, you trust it
  • No formal permission system
  • Security through code review (MIT, open source)
  • Community skills are user-vetted

Claudie's approach

  • Own accounts (claudie@every.to) — not mirroring someone's full access
  • PreToolUse hooks intercept dangerous tool calls
  • Ring-based access control (4 tiers)
  • Identity gates reject unknown Slack IDs before model spawns
  • dontAsk permission mode for browsing jobs
The honest assessment: Cloud-sandboxed agents are structurally more secure. Our compensating controls are sophisticated, but they're defense-in-depth on top of an inherently riskier architecture. We accept that tradeoff because we need capabilities (authenticated browsing, local scheduling, proactive messaging) that sandboxed platforms can't provide. For most enterprises, the cloud platforms are the right choice.

7. What This Means for Our Clients

The recommendation framework

Use OpenAI Workspace Agents when…

  • The team already lives in ChatGPT
  • You want no-code setup — describe the workflow, connect tools, go
  • The use case is team-shared: triaging requests, preparing reports, responding to messages
  • Slack integration is a requirement
  • You want built-in approval gates without building them yourself
  • You're okay with OpenAI models only

Best for: operations teams, non-technical teams, quick wins

Use Claude Managed Agents when…

  • You need developer-level control over agent behavior
  • Security posture is the top concern (vault-based credentials, container isolation, networking controls)
  • The use case involves long-running execution with many tool calls
  • You want multi-agent orchestration
  • You need the strongest observability (Events API, outcome evaluation)
  • The team has engineering capacity to build the integration layer

Best for: engineering teams, security-conscious enterprises, complex workflows

Use OpenClaw when…

  • Data must never leave your infrastructure
  • You want model flexibility (switch between Claude, GPT, local models)
  • Multi-channel presence matters (WhatsApp, Telegram, Signal, Discord, etc.)
  • You have technical staff who want full control
  • Budget is a constraint (free + API costs)

Best for: technical teams with data sovereignty requirements

Build a custom harness (like Claudie) when…

  • You need a true "AI employee" — proactive, always-on, with institutional memory
  • The use case requires authenticated access to internal systems the agent needs to drive, not just read
  • Scheduling is core to the value (morning briefs, periodic refreshes, monitoring)
  • You're willing to invest 100+ hours in setup and ongoing maintenance
  • The team is small enough to justify one shared agent rather than per-user agents

Best for: small teams with high-value operations that justify the investment

Client-specific implications

For hedge funds and PE firms (Bloomberg, financial clients): Claude Managed Agents is the strongest recommendation. The vault-based credential isolation and networking controls address their security concerns. These teams have engineering capacity. Data residency on Anthropic's servers may be a concern — OpenClaw or a custom harness if so.

For media and operations teams (NYT-type clients): OpenAI Workspace Agents is the fastest path to value. No-code setup, Slack integration, scheduling — these teams don't have (or want) engineering involvement. The "describe your workflow, connect your tools" model matches how they think.

For our training curriculum: We should teach all four. The decision framework above is the slide. Clients will ask "which one should I use?" — the answer depends on their team, their constraints, and what they're trying to automate.

8. Our Take

OpenAI shipped product intuition. Anthropic shipped infrastructure elegance. The market wants both.

Workspace Agents is the first cloud platform to ship the full "AI employee" surface: always-on, scheduled, team-shared, in Slack, with approval gates. It's not the most technically sophisticated option — Claude Managed Agents has better credential isolation, multi-agent orchestration, and observability. But it ships the thing people actually want to buy: an agent that shows up in Slack and does recurring work without being asked.

This puts pressure on Anthropic. Managed Agents is architecturally stronger but has a critical gap: no scheduling, no triggers, no proactive operation. Every session must be started by an API call. That means someone (or something) upstream has to decide when to wake the agent up. OpenAI just eliminated that problem at the product layer.

What we'd watch

  • Anthropic adding scheduling to Managed Agents. This is now urgent. OpenAI just showed that scheduling + triggers + Slack integration is the minimum viable "AI employee" product. If Anthropic ships this, the comparison inverts — Claude Managed Agents would have better security and scheduling.
  • OpenAI Workspace Agents pricing. Free until May 6, credit-based after. If credits are expensive, Managed Agents' transparent $0.08/session-hr may look better.
  • OpenClaw's response. Both cloud platforms just validated the category OpenClaw created. But they also made the "why would I self-host?" question harder to answer for non-technical teams.
  • Whether Workspace Agents can handle complex, multi-step workflows reliably. Research preview = early. Codex is powerful but the no-code "describe your workflow" setup may struggle with edge cases that require developer-level precision.

For Claudie specifically

Workspace Agents is the closest thing to what we built — and it validates the bet. An always-on agent in Slack that handles recurring work for a team is clearly what the market wants. We got there first because we built it ourselves.

But we're still in the 20% that needs the full machine. Authenticated Chrome browsing, 12 launchd jobs, email watching, proactive cross-thread messaging, macOS Keychain integration, self-spawning Claude sessions, semantic search over institutional memory — none of the cloud platforms can do all of this today. The gap is closing, though. OpenAI is closing it from the product side (scheduling, Slack). Anthropic is closing it from the infrastructure side (containers, security). Give it a year.
