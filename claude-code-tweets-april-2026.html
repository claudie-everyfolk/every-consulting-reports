How to Use Claude Cowork Like a Second Employee

Jesse Pujji · @jspujji

Reframes AI setup from “prompting” to “management.” Treat Cowork as a new hire — write onboarding docs (who-i-am.md, how-i-talk.md), set workspace boundaries, define “done” clearly. The quality of the onboarding packet determines the ceiling.

Key Learnings
  • The “employee onboarding” mental model changes everything — more intuitive for non-technical users than “write a system prompt”
  • Context files beat prompt engineering: invest once in persistent markdown files that compound across sessions
  • Skills = certifications for your AI employee — repeatable, specialized workflows
  • “No more than 5 clarifying questions” prevents analysis-paralysis loops
  • Dedicated workspace folder = blast radius control before you trust the output

This is the exact frame we should use in bootcamps. “Onboard your AI” lands better than “write prompts.”

How to Set Up Claude So It Never Forgets You

Ruben Hassid · @rubenhassid

“Text files over prompts” — take everything you know and put it in .md files. CLAUDE.md is the sleeper feature. Abandon prompt libraries entirely. Simple first-prompt template: “Read the uploaded files completely before responding.”

Key Learnings
  • CLAUDE.md is “solid and underused by beginners” — the auto-maintained memory file most users never discover
  • The investment should go into markdown documentation, not prompt libraries
  • “Claude now knows you like an old coworker” — scoped memory improves over time
  • First-prompt template (“Read files, then ask clarifying questions”) prevents generic output

Solid advice but not novel for our team. However, Hassid’s consumer-friendly framing makes it good reference material for non-technical bootcamp audiences.

Three Blocks That Make Claude Get Smarter Every Session

Pawel Huryn · @PawelHuryn

Three CLAUDE.md blocks that create a self-improving AI. Knowledge Architecture uses a hypothesis-to-rule promotion system (confirm 3x to promote, contradict to demote). Decision Journal logs reasoning with alternatives. Quality Gate provides objective evaluation criteria. After one month, 24 autonomous rules emerged.

Key Learnings
  • Hypothesis-to-rule promotion is genuinely novel — scientific method applied to AI memory
  • 24 project-specific rules emerged organically in one month without explicit prompting
  • Decision Journal with 3+ alternatives had 80% success rate vs. overconfident quick decisions
  • Quality Gates fix the #1 failure mode: “agents confidently praise their own mediocre work”
  • The distinction: this is “memory with reflection,” not just memory — compounding comes from the reflection loop

Most directly applicable for advanced clients. The hypothesis-promotion system is a teachable “paradigm” — exactly the kind of thing we deliver.

Claude Code Memory Architecture Deep Dive

Himanshu · @himanshustwts

Source-code-level analysis of Claude Code’s memory internals. Three-layer system: Auto Memory (persistent), Session Memory (intra-session, 12K cap), Extract Memories (background agent). MEMORY.md is a 200-line pointer index, not a data store. AutoDream consolidates during inactivity.

Key Learnings
  • Three-layer memory architecture operates at different timescales (persistent, session, background extraction)
  • MEMORY.md is an index of pointers capped at 200 lines — the memory system is a search index, not a database
  • AutoDream runs four phases during inactivity: Orient, Gather, Consolidate, Prune — Claude literally “sleeps on it”
  • Four-type memory taxonomy: User, Feedback, Project, Reference — each drives different retrieval relevance
  • 36.8GB memory leak from 292 agents led to hard architectural caps — multi-agent safety learned through production pain

Ground truth from leaked source code. When clients ask “how does Claude remember?” we can explain the real architecture. AutoDream is a powerful metaphor for long-running deployments.

The Anthropic Labs Insider Story

Boris Cherny · @bcherny — Head of Claude Code

Head of Claude Code reveals the tiny Anthropic Labs team shipped MCP, Skills, Claude Desktop, and Claude Code. Plain glob+grep beat vector databases for context retrieval. Boris ships 20–30 PRs/day via 5 parallel Claude instances. Prototypes replaced PRDs entirely.

Key Learnings
  • “Search over retrieval” beat every fancy approach — plain glob and grep, driven by the model, outperformed vector databases
  • 20–30 PRs/day workflow: 5 parallel instances, start in plan mode, iterate, then one-shot execution
  • “Prototypes over PRDs” killed the old product process — they couldn’t have shipped this with static mocks
  • Cowork was built in ~10 days and grew faster than Claude Code at launch — non-technical users were the bigger market
  • Meta-skill shifted from “deep work” to “context switching” — orchestrating multiple AI instances
  • “Update your CLAUDE.md after every correction” — simplest possible compounding practice

The most authoritative source in the batch. “Search over retrieval” alone should change how we advise clients on RAG. The parallel-instances workflow is the most concrete productivity framework.

What This Tells Us

The single most actionable insight: the shift from “prompting” to “onboarding” (Pujji) combined with “update CLAUDE.md after every correction” (Cherny). These two practices together create compounding AI capability with minimal effort. One reframes the relationship; the other gives it a ratchet. Neither requires technical sophistication — both are teachable in a single bootcamp session.

Five non-obvious insights across all sources

  1. Memory is architecture, not feature. Claude Code runs a three-layer system with background consolidation (AutoDream). Clients who treat memory as “just context” will plateau; those who design for it will compound.
  2. Search beats retrieval. Boris Cherny confirmed that plain glob+grep, directed by the model, outperforms vector databases for context retrieval. This directly challenges the RAG-first advice most clients receive.
  3. The hypothesis-promotion pattern is a new primitive. Huryn’s “confirm 3x to promote, contradict to demote” system is a teachable paradigm — scientific method applied to AI memory. No one else is teaching this.
  4. The meta-skill is now orchestration, not execution. Boris runs 5 parallel Claude instances. The bottleneck shifted from “can you code fast” to “can you context-switch and manage multiple agents.” This has training implications.
  5. The platform shift happened in 4 weeks. Ejaaz’s catalog shows Claude Code crossed from developer tool to knowledge-worker platform in a single shipping sprint. Clients who evaluated it even two months ago saw a fundamentally different product.
